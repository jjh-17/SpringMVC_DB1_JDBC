[트랜잭션]
Atomic - 원자성. 트랜잭션 내 모든 작업은 모두 성공 혹은 실패 밖에 없어야한다.
Consistency - 모든 트랜잭션은 일관성있는 DB 상태 유지
Isolation - 동시 실행되는 트랜잭션이 서로 영향을 미치지 않도록 한다.
Durability - 트랜적션 결과는 항상 기록되어야 하며, 실패하더라도 성공한 트랜잭션 내용을 복구해야 한다.

[트랜잭션 격리 수준]
Read Uncommited - 커밋되지 않은 읽기
Read Commited - 커밋된 읽기
Repeatable Read - 반복 가능한 읽기
Serialization - 직렬화 가능
