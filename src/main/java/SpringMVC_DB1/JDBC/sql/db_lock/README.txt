[DB 락]
-세션1에서 데이터를 수정하는 동안 세션2에서 동일한 데이터를 수정할 경우, 원자성에 위배됨
-세션1에서의 데이터 수정이 완료되지 않은 상태에서 세션2가 롤백을 할 경우, 세션1은 잘못된 데이터를 수정하게 된다.
    ==> 커밋/롤백 전까지 다른 세션에서 해당 데이터의 수정을 막을 필요 있음

<DB 락 과정>
1. 세션1의 트랜잭션 시작
2. 세션1이 A의 id에 대한 락을 획득. 업데이트 수행
3. 세션2의 트랜잭션 시작
4. 세션2가 A의 id에 대한 업데이트 시도
    ==> 락 획득 대기(세션1이 락을 가지고 있음). 대기 시간을 넘어서면 '락 타임아웃' 오류 발생
5. 세션1의 커밋 ==> A의 id에 대한 락 반납
6. 세션2가 락을 획득하여 업데이트 수행

[조회 시점의 락]
-데이터를 조회한 시점에 락을 획득할 경우, 'select from update' 수행
